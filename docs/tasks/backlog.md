# Архитектурный backlog

Задачи отсортированы по убыванию приоритета внутри каждого раздела. Выполненные задачи перенесены в конец документа.

## P0 — корректность HTTP-запросов

- [ ] Обрабатывать ошибки сериализации тела запроса через `JSON_THROW_ON_ERROR` и исключение пакета вместо неявной передачи `false` из `json_encode()`
- [ ] Убрать определение `Retryable` через `debug_backtrace()` и Reflection: передавать метаданные безопасности операции в retry-политику явно
- [ ] Определить политику retry для timeout, DNS, connection reset, `502` и `503`, сохранив безусловный запрет повтора для `NonRetryable`
- [ ] Учитывать `Retry-After` и `X-Ratelimit-Retry`: поддержать секунды и HTTP-дату, ограничить максимальную задержку и использовать локальную задержку только как fallback

## P1 — retry и устойчивость

- [ ] Вынести retry в отдельную `RetryPolicy`, чтобы настраивать коды, транспортные ошибки, методы, количество попыток и задержки независимо от endpoint
- [ ] Добавить exponential backoff и jitter только для случаев, когда сервер не передал обязательную задержку
- [ ] Абстрагировать блокирующий `usleep()` через `Sleeper` для тестирования и возможности альтернативной реализации
- [ ] Покрыть retry для `Retry-After`, `X-Ratelimit-Retry`, timeout, DNS, connection reset, `502`, `503` и исчерпания попыток

## P1 — границы ответственности и контракты

- [ ] Ввести интерфейс `HttpTransport` и внедрять его через конструктор вместо создания `new Client()` внутри `AbstractEndpoint`
- [ ] Передавать исполнитель запросов в endpoint и subpoint через композицию, не открывая transport-методы как часть пользовательского API
- [ ] Разделить создание клиента, проверку токена, retry, интерпретацию ошибок и хранение последнего ответа между отдельными компонентами
- [ ] Заменить неявный extension point `method_exists($this, 'middleware')` объявленным методом или интерфейсом

## P1 — подготовка версии 5.0.0

- [ ] Проверить совместимость dev-зависимостей с заявленным PHP 8.1: `phpunit/phpunit:^12` требует более новую версию PHP
- [ ] Добавить CI-матрицу PHP 8.1–8.4 с установкой lowest и highest версий зависимостей
- [ ] Подготовить migration guide для breaking changes 5.0.0: `ApiResponse`, исключения, retry, удаление mutable-свойств `Client` и `AbstractEndpoint::__call()`
- [ ] Зафиксировать публичный API 5.0.0 тестами обратной совместимости или инструментом проверки API/BC

## P2 — модель ответа и типизация

- [ ] Добавить точные возвращаемые типы или DTO вместо распространённого `mixed`, неструктурированных `object` и массивов
- [ ] Пометить `responseCode()`, `response()`, `responseHeaders()`, `rawResponse()` и `responseRate()` deprecated с планом удаления в следующей major-версии
- [ ] Типизировать события listeners отдельными DTO вместо неструктурированных массивов
- [ ] Добавить интерфейс PSR-3 logger вместо прямого вызова `error_log()` для ошибок listeners

## P2 — тестирование

- [ ] Покрыть ошибки сериализации запроса, malformed JSON ответа и некорректные значения rate-limit заголовков
- [ ] Проверить, что `NonRetryable` запрещает повтор при каждом поддерживаемом HTTP- и транспортном сбое
- [ ] Добавить unit-тесты совместимых accessor последнего ответа и `ApiClientException::response()`

## P2 — статический анализ

- [ ] Поднять PHPStan для `src` с уровня 2 до уровня 4, исправив типы коллекций, параметров и возвращаемых значений
- [ ] Поднять PHPStan для `src` с уровня 4 до уровня 6 после типизации endpoint и DTO
- [ ] Анализировать старые integration-тесты отдельно от `src`, постепенно добавляя `void`, типы параметров и непустые проверки

## P3 — сопровождение

- [ ] Привести именование свойств к единому стилю, включая `$Client` и `$Adv`
- [ ] Настроить единый стиль окончаний строк, чтобы исключить постоянные предупреждения LF/CRLF в Git
