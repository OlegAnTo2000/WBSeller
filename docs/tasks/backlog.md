# Архитектурный backlog

Задачи отсортированы по убыванию приоритета внутри каждого раздела.

## P0 — корректность HTTP-запросов

- [x] Унифицировать контракт `Client::request()`: одинаково обрабатывать HTTP-ошибки с JSON- и не-JSON-телом, не смешивать возврат результата с выбрасыванием Guzzle-исключений
- [x] Ограничить автоматический retry идемпотентными операциями, чтобы повтор POST, PATCH или DELETE после 504 не создавал дублирующие изменения
- [x] Нормализовать транспортные и HTTP-ошибки в исключения пакета, чтобы наружу не выходили непредсказуемые типы исключений Guzzle
- [ ] Исправить вызов отсутствующего `Marketplace::getOrdersClient()` в `Subpoint/CrossBorder.php`

## P1 — retry и устойчивость

- [ ] Учитывать заголовки `Retry-After` и `X-Ratelimit-Retry` вместо одной фиксированной задержки
- [ ] Добавить exponential backoff и jitter для предотвращения одновременных повторов запросов
- [x] Валидировать параметры `retryOnTooManyRequests()`: количество попыток должно быть не меньше 1, задержка — неотрицательной
- [ ] Вынести retry в отдельную `RetryPolicy`, чтобы настраивать коды, методы и лимиты независимо от endpoint
- [ ] Абстрагировать блокирующий `usleep()` через `Sleeper` для тестирования и возможности альтернативной реализации

## P1 — границы ответственности и контракты

- [ ] Убрать `AbstractEndpoint::__call()`, который открывает protected HTTP-методы и скрывает реальный публичный контракт от PHPStan и IDE
- [ ] Ввести интерфейс исполнителя запросов и передавать его endpoint/subpoint-классам через композицию
- [ ] Внедрять HTTP-транспорт через конструктор вместо создания `new Client()` внутри `AbstractEndpoint`
- [ ] Разделить создание клиента, проверку токена, retry, интерпретацию ошибок и хранение ответа между отдельными компонентами
- [ ] Заменить неявный extension point `method_exists($this, 'middleware')` объявленным методом или интерфейсом

## P2 — модель ответа и состояние

- [ ] Ввести объект ответа с телом, HTTP-статусом, заголовками и rate-limit вместо набора mutable-свойств последнего запроса
- [ ] Устранить зависимость endpoint-методов от глобального состояния `responseCode()`, `response()` и других данных последнего запроса
- [ ] Добавить точные возвращаемые типы или DTO вместо распространённого `mixed`, неструктурированных `object` и массивов
- [x] Определить единый формат декодирования JSON, включая массивы, скалярные значения, пустое тело и невалидный JSON

## P2 — валидация и конфигурация

- [x] Пересмотреть fail-open проверку JWT: токен JWT-формата, который не удалось разобрать, сейчас отклоняется до запроса
- [x] Разделить локальную проверку claims токена и серверную аутентификацию, явно задокументировать границы такой проверки
- [x] Валидировать listeners и middlewares при конфигурации вместо молчаливого игнорирования некорректных значений
- [ ] Не подавлять исключения listeners без следа: предусмотреть логирование или отдельный обработчик ошибок наблюдаемости
- [x] Заменить строковые имена API и HTTP-методов типизированными enum с сохранением строкового публичного API

## P2 — тестирование

- [x] Добавить unit-тесты транспорта через Guzzle `MockHandler`, не требующие реального API-ключа
- [x] Покрыть одинаковую обработку JSON-, text/plain- и пустых тел для 400, 401, 429, 500 и 504
- [ ] Проверить количество попыток и отсутствие retry по умолчанию
- [x] Проверить запрет автоматического retry для неидемпотентных операций
- [ ] Покрыть `Retry-After`, `X-Ratelimit-Retry`, сетевые timeout, DNS-ошибки и исчерпание попыток
- [x] Проверить регистрацию, порядок выполнения и ошибки middleware/listeners
- [x] Отделить интеграционные тесты с реальным WB API от быстрого unit-набора

## P3 — статический анализ и сопровождение

- [ ] Устранить текущие ошибки PHPStan, в первую очередь вызовы protected/отсутствующих методов и несовпадения сигнатур
- [x] Актуализировать устаревшие тесты, которые вызывают отсутствующие методы endpoint-классов
- [x] Исправить несовпадения PHPDoc и нативных типов в `Chat`, `Content`, `Marketplace` и `Questions`
- [ ] Поднять уровень PHPStan с 2 как минимум до 6 после устранения базовых ошибок
- [x] Синхронизировать комментарии `Client` с фактическими настройками SSL, обработкой ошибок и retry
- [ ] Привести именование свойств к единому стилю, включая `$Client` и `$Adv`
